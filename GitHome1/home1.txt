First level: 1. Как скопировать на свой компьютер репозиторий из github.com, что должно быть сделанно предварительно?
Создается новый репозиторий в Git. 
КОпируется в репозитории адрес (через Kode, SSH).
После этого зайти на github, проверить свое местонахождение и/или указать адрес необходимой папки, в которую необходимо скопировать репозиторий.
Таким образом, в строке будет указан адрес необходимой папки, после этого прописывается команда git clone и вставляется скопированный в репозитории адрес.
Командой git remote -v подключаемся к репозиторию
ПОсле чего в созданной папке создается файл, в который вносятся соответствующие изменения через редактор (например, nano).
Изменения сохраняются.
После чего данный файл необходимо добавить в репозиторий 
команда git add и название файла - подготовка к коммиту 
Коммит git commit -m "комментарий" название файла
git push - отправка файла в репозиторий




Как создать git репозиторий из локального каталога на своем комьютере. Как его отправить на github.com?
создается папка (через mkdir или непосредственно в папке на компьютере)
создаем туда файл
делаем команду git init
добавляем файл через git add название файла (или .)
делаем коммит  git commit -m "комментарий" файл
git remote -v  - подключение к git
заходим на сайт github.com. 
создаем новый репозиторий
копируем первую строку из абзаца " ...or push"
вставляем в gitbush
делаем remote -v
опять в git копируем вторую строку, вставляем в gitbush
делаем remote -v
возвращаемся в git копируем третью строку, вставляем в gitbush

Какую информацию нам дают команда :
git status
Команда git status показывает состояния файлов в рабочем каталоге и индексе: какие файлы изменены, но не добавлены в индекс; какие ожидают коммита в индексе.

git log

По умолчанию (без аргументов) git log перечисляет коммиты, сделанные в репозитории в обратном к хронологическому порядке — последние коммиты находятся вверху.
